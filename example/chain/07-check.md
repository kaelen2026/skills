# 07 · check：发布前对抗式自审

> skill：`check` —— 对自己的 diff / PR / release 做对抗式自审与安全门禁。
> 触发：「合并前看看代码」

check 站在"挑刺的资深 reviewer"立场过 diff，分等级，安全门禁优先。下面是这次对 `app/` 全量 diff 的自审。

## 安全门禁（先过这道）

| 项 | 结论 |
| --- | --- |
| 是否引入随机/时间等不确定源到核心路径 | ✓ 已清（hunt 移除了随机 salt，回归测试钉死） |
| 是否信任未校验的外部输入 | ⚠ 见下 M1 |
| 是否有命令注入 / 路径穿越 | `URLSHORT_DB` 是本地路径、`execFileSync` 用数组传参不过 shell，无注入面 |
| 是否泄露密钥 | 无密钥 |

## 发现

**M1（中）· expand 无条件信任存储内容。**
`expand` 直接 `JSON.parse(readFileSync(path))` 后返回值。如果 `links.json` 被手工改坏成非法 JSON，`load` 会抛未捕获异常，CLI 直接崩栈。

- 评估：自用工具、文件自己管，危害有限，但崩栈对"自己半年后"不友好。
- 处置：记为已知限制（不阻塞本次发布），如复发再加 `try/catch` 给可读报错。属 scope-guard 边界外的"够用即可"。

**L1（低）· `list` 输出用 `\t` 分隔，URL 含 tab 时对齐会乱。**

- 评估：URL 几乎不含 tab，纯显示问题，不修。

**L2（低）· `dbPath()` 缺省路径写在 app 目录内。**

- 多用户/只读安装场景会写不进去。本工具自用单机，可接受。记一笔。

## 没有发现的（对抗式确认过）

- 短码生成路径已是纯函数，无隐藏状态（hunt 的修复 + 回归测试覆盖）。
- `shorten` 重复缩同一 URL 只会覆写同一 key，不会让存储膨胀。
- BigInt 路径无 Number 精度丢失。

## 裁决

**放行**。M1/L1/L2 都属"够用即可"范围，已在 [decision-log](04-decision-log.md) 精神下记为已知限制，不阻塞。核心正确性（确定性）有回归测试守门。

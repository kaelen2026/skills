# Logic 原型：在终端里验逻辑手感

何时加载：问题是"这个状态机/业务流程/算法的手感对不对"，需要亲手把状态跑出来感受。

## 目标

把要验的逻辑包成一个交互式终端小程序：你输入一个动作，它执行转移，然后把转移后的**完整状态**打印出来。重复，直到你判断这个设计的手感是对还是错。

## 搭法

1. **用项目现有运行时，一条命令启动。** Node 项目就 `node prototype-checkout.mjs` / `tsx`，Python 就 `python proto_checkout.py`，Rust 就放进 `examples/`。不引新工具。
2. **文件名标临时。** 如 `PROTOTYPE-checkout-flow.mjs`，放在它要验的模块旁边。
3. **核心循环极简：**
   - 初始化内存里的状态（写死的假数据即可）。
   - 读一个动作（命令行参数、stdin 一行、或简单菜单）。
   - 把动作喂给你要验的那段逻辑。
   - 打印**完整**状态，不只是 diff。让每个字段都可见。
   - 回到读动作。
4. **状态全在内存。** 不连 DB、不写文件、不留副作用。重启就回到初始。

## 要点

- 逻辑本身用真实代码（你要验的就是它），外面的壳越糙越好。
- 打印状态要全：藏起来的字段正是手感不对的来源。
- 跳过错误处理，除非崩溃挡住了你跑流程。
- 验完把结论（这个状态机设计行不行、哪里别扭）写进 commit/ADR/笔记，再删。

# UI 原型：并排视觉变体验"该长什么样"

何时加载：问题是"这东西该长什么样"，需要把几个视觉方案放一起对比。

## 目标

在**同一条路由**上放几个视觉变体，用 URL 参数切换（如 `/checkout?v=1`、`?v=2`、`?v=3`），直接肉眼对比哪个对。不接真实数据，用写死的假数据喂满布局。

## 搭法

1. **复用项目现有的前端工具链，一条命令启动。** 项目跑 Vite/Next 就开它的 dev server，别为原型搭新构建。
2. **一条临时路由承载所有变体。** 读 URL 参数选渲染哪个变体；缺省渲染变体 1。文件名/注释标 `PROTOTYPE`。
3. **每个变体是一段独立的渲染。** 不抽共享组件、不做配置化，直接各写各的 markup，重复无所谓，原型不为复用而生。
4. **数据写死。** 用代表性的假数据（含长文本、空态、极端值这些会暴露布局问题的情况），不接 API、不接 DB。

## 要点

- 变体之间差异要明显，是在做选择不是微调；微调留给真正实现阶段。
- 故意塞边界数据（超长名字、0 条、99+），布局问题在这些地方才现形。
- 跳过交互逻辑、表单校验、状态管理，除非要验的正是某个交互手感。
- 选定后把结论（选了哪个变体、为什么）写进 commit/ADR/笔记，删掉原型路由，再按正式标准重新实现选中的方案。

# 搜索意图与 on-page 优化

排名的前提是给搜索者他真正想要的。先判意图，再看 SERP 现状，最后做 on-page。凭感觉堆关键词是这一层最大的失败模式。

## 1. 搜索意图分类

搜同一个词的人，想要的东西不同。先判这个目标查询属于哪类，页面就该是哪类：

- **信息型（informational）**："怎么做 X""X 是什么""X 教程"。想要答案、步骤、解释。页面给清晰的内容、结构化的步骤，别一上来就推销。
- **导航型（navigational）**："某产品名""某品牌 登录"。想去某个特定站点。优化品牌词、确保官网是第一结果。
- **交易型（transactional）**："买 X""X 价格""X 下载"。想完成动作。页面给清晰的 CTA、价格、购买/下载路径。
- **调研型（commercial investigation）**："X 对比 Y""最好的 X""X 评测"。买前调研。页面给对比、优缺点、适用场景。

意图判断错，内容类型就错。给信息型查询塞一个落地页推销，排不上也留不住人。

## 2. 读 SERP 现状

判完意图，去搜一下目标词，看排在前面的是什么 —— 这是搜索引擎已经认定的"这个词该给什么"。

- **页面类型**：前排是教程、对比、产品页还是论坛帖？你的页面类型要对得上，否则是在错误的赛道竞争。
- **内容深度与角度**：前排覆盖了哪些点、什么角度？找它们都没覆盖好的角度切入（信息缺口），别做又一个雷同页。
- **SERP 特性**：有没有精选摘要、People Also Ask、图片包、视频位？这些决定你该用什么内容形态去争（如 FAQ schema 争 PAA）。
- **现实判断**：前排全是高权重大站，新站短期争不过 —— 换更长尾、竞争更低的变体（回到 SKILL.md 的"先赢可赢的词"）。

## 3. 选词（针对独立开发者）

- **优先长尾**：3-5 个词的具体查询，竞争低、意图明确、转化高。"claude code 配置 mcp" 比 "claude code" 好赢得多。
- **要有真实需求依据**：能说清"谁、在什么场景下、会这么搜"。说不清就不是好目标词。
- **一页一主查询**：每个页面主攻一个查询及其近义变体，别一页想吃十个不相关的词。

## 4. 标题（title tag）

- 含主查询，放前面，自然语言，给人看。
- 唯一、不超过约 60 字符（中文更短）避免被截。
- 别堆关键词、别标题党。点进即跳出会掉排名。
- 模板：`主查询核心 - 关键限定 | 站名`，例：`Claude Code 配置 MCP 服务器 - 完整步骤 | 站名`。

## 5. meta description

- 不直接决定排名，但决定 SERP 上的点击率；缺了搜索引擎会随便截首段。
- 一两句，对齐意图，包含主查询（命中会加粗），自然地给一个点进来的理由。
- 唯一，约 150 字符内（中文更短）。

## 6. 正文 on-page

- **首屏直接回应意图**：信息型就先给答案/步骤摘要，别让人滚半天。
- **标题层级**：一个 h1 含主题，h2/h3 拆子问题，顺带覆盖 People Also Ask 里的相关问题。
- **覆盖信息缺口**：补 SERP 前排没讲好的那部分，这是新站能赢的地方。
- **内链**：用描述性锚文本连到相关页面，把上下文和权重串起来。
- **结构化数据配合**：教程配 HowTo、问答配 FAQPage，且只反映页面真实可见内容。

## 7. 验收

优化完自检：搜目标词，想象自己是那个搜索者 —— 这个标题会让我点吗？点进来首屏给到我要的了吗？还是又一个雷同页？答不上"我为什么排得上前排没给到的东西"，就还没做完。

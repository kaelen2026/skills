# 窗口解析与对齐

复盘的时间窗口决定了所有 git 查询的边界。算错边界，整份复盘的数字都偏。

## 参数

- `retro`（无参数）= 最近 7 天
- `retro 24h` = 最近 24 小时
- `retro 14d` = 最近 14 天
- `retro 30d` = 最近 30 天
- `retro 7w` 等 = 最近 N 周（按 N×7 天算）
- `retro compare` = 本窗口（默认 7d）vs 紧邻的上一个等长窗口
- `retro compare 14d` = 显式窗口的对比

参数不匹配 `数字+d/h/w` 或 `compare [窗口]` 时，打印用法并停下，不要猜。

## 午夜对齐（关键）

对 day/week 单位，要算出一个绝对的本地午夜起始日期，不要用相对字符串。

原因：`git log --since="7 days ago"` 在晚上 11 点跑，起点是 7 天前的 23:00，不是那天的 00:00。会漏掉那天白天的 commit。

做法：算出起始日历日期，显式加 `T00:00:00`。

例：今天是 2026-06-17，窗口 7d，则起始日期是 2026-06-10，查询用 `--since="2026-06-10T00:00:00"`。

week 单位先乘 7 转成天（`2w` = 14 天回溯）。

hour 单位不适用午夜对齐，直接用 `--since="N hours ago"`。

所有时间按系统本地时区报告，不要设 `TZ`。

## compare 模式的两个窗口

1. 当前窗口：用午夜对齐的起始日期，逻辑同上。
2. 上一个等长窗口：同时用 `--since` 和 `--until`（都午夜对齐）避免重叠。

例：7d 窗口当前起始 2026-06-10，则上一个窗口是 `--since="2026-06-03T00:00:00" --until="2026-06-10T00:00:00"`。

输出并排对比表，带 delta 和箭头，再写一段简短叙述点出最大的改善与退步。只持久化当前窗口的快照（见 snapshot.md），不存上一个窗口的指标。

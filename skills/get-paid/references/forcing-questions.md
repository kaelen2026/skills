# 变现逼问 · 全文与追问

何时加载：跑核心逼问、需要把"没人付钱"推到具体那一层时。

## 按现状路由（不必全问）

- 还没接收费 → Q1, Q2, Q5（先确认有没有人会付、价值锚在哪）
- 有收费但零付费 → Q1, Q2, Q4
- 有付费但转化低 → Q2, Q4, Q5
- 有付费但留不住（churn 高） → Q3 优先，先修留存再谈转化

目标适配：开源加赞助把"付费"重述为"赞助/支持"，但 Q1（有几个人真出过钱）和 Q2（锚在什么价值）照问。

## 怎么推（软探索 vs 严格诊断）

- "等用户量再大点就收费" → 后推陷阱："用户量大不会自动变成付费，免费用户和付费用户常常是两群人。现在这批里，哪个具体的人会因为什么具体场景先付钱？"
- "我觉得定价太高了" → 价值测试："嫌贵通常不是价格问题，是没让用户看到值这个价。把价格锚在他因此省下/赚到的东西上，再看还嫌不嫌贵。"
- "大家都说很好用" → 付费现实："好评是免费的。这些说好用的人里，有几个掏过信用卡？说好用却不付钱，往往说明它是 nice to have 不是 must have。"
- "对标某产品定了个价" → 锚点测试："抄来的价格锚在别人的价值上，不是你的。你的用户用你这个，具体省下/赚到了什么？说不出就是拍脑袋。"
- "免费版就够好了" → 边界测试："如果免费版好到没人需要升级，那不是慷慨，是没给付费一个非买不可的理由。付费墙划在哪个真实需求上？"

## Q1：付费现实

**问**："现在有几个人付过真钱？他们有什么共同点（什么场景、什么职位、为什么是他们先付）？"
**推到听见**：一个具体数字 + 付费者画像。一个都没有，先别谈转化率，先谈"第一个付费用户长什么样"。
**红旗**："还没正式收费但很多人感兴趣""问卷里 70% 说愿意付"。兴趣和问卷不是付费，只认掏过卡的行为。

## Q2：价值锚点

**问**："你的价格锚在用户因此省下/赚到/避免损失的什么上？给个具体数量。"
**推到听见**：一个可量化的价值（每周省 5 小时、每月多赚 X、避免一次 Y 损失）。价格是这个价值的一个零头才好卖。
**红旗**："按成本加了百分之多少""看竞品定的"。锚在成本或竞品上 = 没锚在用户价值上，难卖是必然。

## Q3：留存真相

**问**："付费（或重度使用）用户第二个月还在吗？流失的人为什么走？"
**推到听见**：一个真实的留存/续费数字 + 流失原因。churn 高就说明问题在留存，不在转化。
**红旗**："只看了总注册""没追踪留存"。不看留存就优化拉新和转化，是往漏桶里加水。
**判据**：留存明显不健康时，本轮诊断到此为止，先指回留存，别给出任何拉高转化的建议。

## Q4：卡点定位

**问**："卡在没人付，到底是嫌贵（定价）、不觉得值（价值）、还是根本没走到付费那一刻/不知道能付费（触达）？"
**推到听见**：用户行为证据指向其中一层。三者解法完全不同，混在一起就什么都修不动。
**红旗**："可能都有点吧"。三层一起认 = 没定位。逼到一个主漏点：用户是看到价格走了，还是没看到价值，还是压根没走到付费页？

## Q5：免费边界

**问**："免费版和付费版的线划在哪？一个真有需求的人，会在什么时刻撞到这条线、非升级不可？"
**推到听见**：一个具体的升级触发点（用量到顶、要协作、要导出、要去水印）。这个点要正好卡在真实需求上。
**红旗**："免费版功能挺全的""怕限制太多用户跑了"。免费版好到没痛点 = 没人需要付费，慷慨成了变现的最大障碍。

# Design Reference: 数据可视化

仅当表面是仪表盘、分析视图、图表密集界面或任何数字密集显示时加载。营销页、落地页、通用组件不需要它。

## 仪表盘默认值

仪表盘是实用表面：给用户定位、显状态、可操作。没有 hero、没有营销文案。每个元素都得回答用户的一个问题来挣得位置。

- 主布局：状态摘要在顶、明细在下；或 sidebar 过滤器 + 主图表区。
- 留白：比营销页紧。用户在扫不在读。用慷慨的列间距，不是慷慨的行高。
- 数字密度：屏上一次很多数字不是问题，没对齐的拥挤才是。所有数字列用 `font-variant-numeric: tabular-nums`。数字右对齐，标签左对齐。

## 图表选型

| 用途 | 图表类型 |
| --- | --- |
| 跨类目比较值 | 柱状图（标签长用横向） |
| 随时间趋势 | 折线图；多点时间序列别用柱 |
| 部分-整体关系 | treemap（6+ 段）或堆叠柱；饼图只用 2-4 段 |
| 分布 | 直方图或箱线图；绝不用饼图 |
| 相关性 | 散点图；不用折线 |

超过 4 段的饼图什么都没传达。用 treemap 或排序列表代替。

## 数字密集界面

- 每个数字列用 `font-variant-numeric: tabular-nums` 让数位垂直对齐。
- 所有数字右对齐，所有文本标签左对齐。同一列里混对齐永远是错的。
- 细微行分隔：`1px` 线 `rgba(0,0,0,0.06)`（浅）或 `rgba(255,255,255,0.05)`（暗）。只有表很宽（12+ 列）才用交替行背景。
- 列间距：相邻列至少 `16px`，逻辑上不同的分组之间更多。

## 用产品作对标

用户报某产品作视觉对标（"做得像 Grafana"、"类似 Linear analytics"）时：从那个产品抽 3-5 个数据可视化专属的具体属性，不是泛泛的审美属性。有用的属性：图表配色（精确值）、网格线粗细和透明度、轴标签字号和颜色、tooltip 圆角和阴影、空态处理。不要抽"极简"或"干净"，那不可执行。

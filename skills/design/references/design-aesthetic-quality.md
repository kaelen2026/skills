# Design 审美质量与生产结构

审美评审、给出选项、或落地多页/生产 UI 时加载。独立开发者没有 design lead 帮忙拍板结构，这一节就是那套拍板用的脚手架和交付清单。

## App Shell 规则

做 sidebar + 主工作区布局（Slack、Linear、Notion 类）时：

- 装饰背景默认关
- 表面层级只用背景色台阶和阴影
- 所有交互元素加 `active:scale-95`
- 按钮圆角在每种组件类型内保持一致（选一个：pill、方角、或一个固定值，不混）
- 第一个组件前承诺一个命名圆角刻度（见 design-reference.md 的圆角系统）

## Options Guide

被要求给设计选项时，给至少 3 个跨真正不同维度的变体：

- **要变的维度**：视觉密度、排版个性、色温、布局结构、动效性格、装饰量、抽象程度
- **混合路数**：一个紧贴现有惯例、一个重混品牌 DNA、一个刻意意外
- **从基础到大胆递进**：第一个安全可懂，后面的推得更远
- 只换强调色的三个不算三个变体。变布局、字体、动效、表面处理。

## DESIGN.md Scaffold（可选，生产 UI）

多页或生产 UI，写第一个组件前先发一份简短的 DESIGN.md 式总结。它强制枚举那些否则会被留作隐含的决策，并让用户尽早纠正方向。一个人做时这一步替你扮演那个本该提问的设计评审。九个部分：

1. **视觉主题与氛围**：mood、密度、设计哲学，2-3 句
2. **调色板与角色**：每个颜色 token 的语义名 + 值 + 功能角色
3. **排版规则**：字体家族、字号刻度、字重刻度、行高、字距；超过 4 级用表
4. **组件样式**：按钮（全部态）、卡片（若用）、输入、导航；各自带状态（default、hover、active、disabled）
5. **布局原则**：间距刻度、网格列、留白哲学
6. **深度与抬升**：阴影系统或背景色台阶系统；描述每一级
7. **Do's 和 Don'ts**：5 到 10 条本项目专属的护栏，不是泛用规则
8. **响应式行为**：断点、导航如何收起、触控目标下限
9. **Agent Prompt 指南**：快速颜色参照（名: 值 对）+ 3 到 5 个可直接粘进后续请求的示例组件 prompt。prompt 要具体到能直接执行：每个值、每个圆角、每个字距、每个字重都内联。标准（值仅示意，用项目自己的 token）："在 `{bg-canvas}` 上做 hero，标题 48px 字重 600，行高 1.00，字距 -0.022em，颜色 `{text-primary}`，CTA 用 `{accent}` 配 `{btn-radius}` 圆角"，要这个级别的具体，不是"带主色和 CTA 按钮的 hero"

单组件或快速原型跳过这步，三行论点足够。

## 交付前清单：战略性遗漏

最常从 AI 生成 UI 里漏掉的项，因为它们要的是有意的产品思考而非视觉判断。独立开发者一个人很容易跳过这些，每次交付前都过一遍：

- [ ] **自定义 404 页**：泛用框架 404 是坏体验。做带清晰返回路径的品牌页（首页链接、搜索、或最常用导航项）。
- [ ] **返回导航**：每个用户操作可达的页面都要有清晰可用的返回路径。死胡同页（详情视图、确认屏、纯 modal 流）是 UX 失败。
- [ ] **表单客户端校验**：email 字段提交前校验格式；必填项显示内联错误；错误消息出现在字段旁而非只在表单顶部。
- [ ] **skip-to-content 链接**：文档第一个可聚焦元素是视觉隐藏的 `<a href="#main-content">Skip to main content</a>`。键盘可达性必需。
- [ ] **cookie 同意**：产品在欧盟或加州运营时，cookie 同意 UI 不可选。按司法辖区 scope 实现。
- [ ] **页脚 Privacy 和 Terms 链接**：每个产品页都要。缺失等于宣告"demo"而非"产品"。

这些不是视觉打磨项，是 demo 和可上线产品之间的区别。

## 参考材料优先级

参考 UI 同时有源码和截图时：读源码。源文件含精确 token 值，截图要靠猜。

只有 URL 时：抓取只返回提取的文本，无布局信息。视觉参考请用户给截图，不要从剥离的 HTML 推断。

## 给现有 UI 加东西

扩展现有界面时先理解它的视觉词汇。写第一行新代码前匹配以下全部：

- 文案语气和阅读层级
- 颜色调色板和语义颜色角色
- hover 和 click 态：缩放、变色、下划线、背景填充
- 动画风格：时长、缓动、回弹还是 ease-out
- 阴影和卡片处理
- 布局密度和留白节奏
- 圆角选择

换内容会让新组件显得格格不入，就说明词汇没匹配到位。完整清单在 design-reference.md。
